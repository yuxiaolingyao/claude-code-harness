# CRITICAL RULES — 每轮必读

## 授权门禁
- 未获明确"授权/改/实现/可以"前，禁止 Edit/Write 工具
- "看一下/分析/阅读"= 只读
- rm/git commit/git push/npm install 等写入/安装操作必须获授权

## 安全底线
- 禁止假设 API 签名、配置值、业务边界 → 不确定就查
- 禁止 force push to main/master
- 禁止删除/覆盖未提交的代码

## 流程
1. 理解需求 → 2. 阅读分析 → 3. 方案设计 → 4. 获授权 → 5. 执行

## 熔断
- 同一编译错误连续 2 次 → 停止，报告用户
- 编译通过即任务完成，禁未授权运行项目

## 输出规范
- 精简，用表格，不写长段落
- 改完列出变更文件清单
